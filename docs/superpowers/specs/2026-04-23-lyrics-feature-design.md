# 歌詞機能 設計書

**日付**: 2026-04-23  
**対象プロジェクト**: ICE CREAM MUSIC BOX  

---

## 概要

各楽曲に歌詞を追加し、再生位置に同期して表示する機能。LRCファイルをCloudflare R2で管理し、デプロイなしで歌詞を追加・更新できる運用フローを実現する。

---

## ユーザー体験

### 歌詞ありの曲
1. プレイヤーにピンクの「♪ 歌詞」ボタンが表示される
2. タップするとフルスクリーンモーダルが開く
3. 再生位置に合わせて現在行が大きく白くハイライト（Apple Musicスタイル）
4. 前後の行は小さくグレーで表示
5. 上下にグラデーションフェードで歌詞がスムーズに流れる
6. モーダル下部に再生プログレスバーを表示
7. × ボタンまたは背景タップで閉じる

### 歌詞なしの曲
1. プレイヤーにグレーの「♪ 歌詞準備中」ボタンが表示される
2. タップするとモーダルが開き「歌詞準備中 / もうしばらくお待ちください」を表示

---

## データ設計

### LRCファイル形式

業界標準のLRCフォーマットを採用。

```
[00:12.34]好きなんだ〜
[00:15.00]溶けちゃう前に食べよう
[00:18.50]アイスクリームが大好き
```

### R2バケット構成

```
R2バケット/
  audio/
    track001.wav
    track002.wav
  lyrics/
    track001.lrc   ← ファイルを置くだけで歌詞が有効になる
    track003.lrc   ← 順次追加していく
```

- ファイル名は音声ファイルと同じベース名（拡張子のみ `.lrc`）
- LRCファイルが存在しない曲は「歌詞準備中」表示

### Track型の変更

Track型は変更しない。歌詞ありかどうかは `useLyrics` のフェッチ結果で判定する。

**ボタン表示の決定フロー:**
1. 曲が変わると `useLyrics` がバックグラウンドでLRCをフェッチ
2. `status === "loading"` 中はボタンをグレー非活性で表示
3. `status === "ready"` → ピンク「♪ 歌詞」ボタン
4. `status === "not_found" | "error"` → グレー「♪ 歌詞準備中」ボタン（クリック可、モーダルで準備中表示）

## コンポーネント設計

### `/api/lyrics/[filename]/route.ts`

```
GET /api/lyrics/track001
→ 200: { lines: [{ time: 12.34, text: "好きなんだ〜" }, ...] }
→ 404: { error: "not_found" }
```

- R2から `lyrics/{filename}.lrc` を取得
- LRCをパースして `{ time: number, text: string }[]` に変換
- R2未設定・ファイル未存在の場合は404を返す

### `useLyrics.ts`

```typescript
useLyrics(filename: string, currentTime: number)
→ {
    lines: LrcLine[],      // パース済み歌詞行
    currentIndex: number,  // 現在再生中の行インデックス
    status: "loading" | "ready" | "not_found" | "error"
  }
```

- `filename` 変更時にAPIフェッチ（曲切り替えでリセット）
- `currentTime` 変化時に `currentIndex` を更新（O(n) 線形探索、33曲程度なら十分）

### `LyricsModal.tsx`

**Props:**
```typescript
{
  isOpen: boolean;
  onClose: () => void;
  trackTitle: string;
  filename: string;       // LRC取得に使用
  currentTime: number;    // 再生位置（秒）
  duration: number;       // 曲の長さ（秒）
}
```

**表示ロジック:**
- `status === "not_found"`: 歌詞準備中メッセージ
- `status === "loading"`: スピナーまたはスケルトン
- `status === "ready"`: 歌詞スクロール表示

**スクロール:**
- 現在行が常に画面中央付近に来るよう `scrollIntoView({ behavior: "smooth" })`

**スタイル（Apple Musicスタイル）:**
```
現在行: font-size 大・color #fff・text-shadow(ピンクグロー)・scale 1.05
前後行: font-size 小・color #444〜#555
上下端: グラデーションフェード(黒→透明)
```

## UIデザイン

### 歌詞ボタン（プレイヤー内）

```
歌詞あり: [♪ 歌詞]      ← ピンク背景・白文字・角丸ボタン
歌詞なし: [♪ 歌詞準備中] ← グレー背景・暗い文字・非活性スタイル
```

### モーダル

```
┌─────────────────────────┐
│ ♪ LYRICS  曲名    [×]  │  ← ヘッダー
├─────────────────────────┤
│  ░░░░░░░░░░░░░░░░░░░░░  │  ← フェード（上）
│                         │
│  溶けちゃう前に食べよう  │  ← past（小・グレー）
│  夏の日のアイスクリーム  │  ← past（小・グレー）
│                         │
│   🍦 好きなんだ〜       │  ← active（大・白・グロー）
│                         │
│   もっと食べたいな       │  ← next（小・グレー）
│   アイスクリームが好き   │  ← next（小・グレー）
│                         │
│  ░░░░░░░░░░░░░░░░░░░░░  │  ← フェード（下）
```

- 背景: 黒〜深い紫のグラデーション
- モーダルは画面全体を覆う（`position: fixed, inset: 0`）
- 背景タップで閉じる

---

## 運用フロー

1. LRCエディタ（[LRC Maker](https://lrc-maker.github.io/) など無料ツール）で歌詞を作成
2. R2の `lyrics/` フォルダにアップロード（ファイル名は音声と同じベース名）
3. デプロイ不要 — 即座に反映される

---

## エラーハンドリング

| ケース | 動作 |
|--------|------|
| LRCファイルなし | 「歌詞準備中」表示 |
| R2未設定 | 「歌詞準備中」表示（エラーは出さない） |
| ネットワークエラー | 「歌詞準備中」表示（retryなし） |
| LRCパースエラー | 「歌詞準備中」表示（壊れたファイルは無視） |
